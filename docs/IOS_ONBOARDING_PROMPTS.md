## Welcome Screen

### 主标题
```
欢迎使用你的AI Career Coach
```

### 副标题
```
一个帮你理清思路、找到答案的职业教练
```

### 描述
```
无论你面临什么职业挑战——升职、转行、团队管理、
还是个人成长——我都会陪你探索最适合你的解决方案。

我不是来告诉你"正确答案"，而是帮你找到属于你自己的答案。
```

---

## How It Works

### 第1步：分享你的情况
```
告诉我你面临的具体挑战，越详细越好。

✅ 好的例子：
"我刚升职当经理，发现很难授权，总想自己做完所有事"

❌ 太笼统：
"怎么当好经理？"
```

### 第2步：回答教练问题
```
我会问一些问题帮你深入思考。

不要着急，慢慢回答。这些问题没有"对错"，
只是为了帮你理清思路。
```

### 第3步：探索和行动
```
我们会一起探索不同的可能性，
你会找到最适合你的下一步行动。
```

## In-Chat Prompts

### 当用户回复太简短时

**检测：** 用户回复 < 10个字

**提示：**
```
💡 多告诉我一些细节吧

比如：
- 具体是什么情况？
- 你尝试过什么？
- 你最担心什么？

越具体，我越能帮到你。
```

---

### 当用户问太宽泛问题时

**检测：** 用户问 "如何..." "怎么..." 且没有背景

**提示：**
```
让我更好地理解你的情况：

1. 你现在在什么情境下遇到这个问题？
2. 具体的挑战是什么？
3. 你已经尝试过什么？

比如，不要只说"怎么当好经理？"
而是说"我刚当经理，团队里有个人不服我..."
```

---

### 当对话陷入僵局时

**检测：** 5轮对话没有进展

**提示：**
```
💡 我们可以换个角度：

1. 如果完全不考虑[限制]，你会怎么做？
2. 你的[老板/团队/家人]会怎么看这个问题？
3. 一年后回头看，你觉得什么最重要？

或者，我们可以：
- 回到最开始的问题，重新梳理
- 聊聊你还没有提到的方面
```

---

### 当用户获得insight时

**检测：** 用户说"原来如此""我明白了""对！"

**提示：**
```
🌟 很好的洞察！

现在，你想：
1. 继续深入这个话题？
2. 聊聊怎么行动？
3. 探索其他相关挑战？

或者，我们可以定一个具体的下一步行动。
```

## Empty State

### 当用户不知道说什么时

**主文字：**
```
不知道从哪里开始？没关系。
```

**引导：**
```
试试回答这些问题：

🤔 最近有什么让你困扰的事吗？
📈 你想在未来3个月达成什么？
💪 有什么技能你想提升？
🤝 有什么关系你想改善？

或者直接说："我不知道从哪里开始，
你能问我几个问题吗？"
```

## Error Recovery

### 当用户感到沮丧时

**检测：** 用户说"这没用""我不知道""太难了"

**提示：**
```
我理解这可能让人沮丧。

让我们慢下来：
- 没有压力，这是你的时间
- 没有对错答案
- 我们可以换个角度

你想聊聊为什么觉得困难吗？
或者我们聊点别的？
```

## Implementation Notes

### iOS代码中的使用

```swift
struct OnboardingPrompts {
    static let welcome = """
    欢迎使用你的AI Career Coach
    
    一个帮你理清思路、找到答案的职业教练
    """
    
    static let firstPromptSuggestions = [
        "💼 我在考虑一个职业决定，需要有人帮我分析",
        "👥 我刚当上经理/领导，遇到了一些挑战",
        "💪 我经常怀疑自己，想知道这正常吗",
        "📈 我想提升某个具体技能",
        "🤝 我和老板/同事/团队的关系有些问题",
    ]
    
    static func promptForCategory(_ category: String) -> [String] {
        switch category {
        case "career":
            return [
                "我拿到两个offer，一个大公司稳定，一个创业公司刺激...",
                "我想转行做产品经理，但不知道从哪里开始...",
                // ...
            ]
        // ... other cases
        }
    }
}
```

## A/B Testing Ideas

### 测试不同的onboarding方式

**版本A：引导式**
- 用户选择类别
- 显示示例
- 鼓励修改示例

**版本B：自由式**
- 只有一个大输入框
- 提供提示文字
- 让用户自由发挥

**版本C：对话式**
- AI先问几个问题
- 了解用户背景
- 然后开始coaching

---

**这些prompts应该：**
✅ 简洁明了
✅ 具体实用
✅ 鼓励行动
✅ 降低门槛
✅ 展示价值

**避免：**
❌ 太多文字
❌ 太抽象
❌ 太正式
❌ 太pushy
